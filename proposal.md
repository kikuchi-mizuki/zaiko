# 高島屋15店舗向け 追加発注・在庫照会システム

---

## 📱 システム概要

繁忙期における高島屋15店舗からの**突発的な追加発注対応を効率化**するLINEベースの自動化システムです。

### 現状の課題（7ステップの伝言ゲーム）

現在、追加発注は以下の流れで対応しています：

1. **店舗マネージャー → 本部バイヤー** に確認（または諦める）
2. **本部バイヤー** が在庫表を確認
3. **本部バイヤー → 貴社** に念のため確認
4. **貴社** が在庫表を再確認（加工会社からの最新情報が未反映の可能性）
5. **貴社 → 本部バイヤー** に連絡
6. **本部バイヤー → 店舗マネージャー** に連絡
7. **貴社 → 加工会社** に発送依頼（納品書作成、納品一覧更新）

**問題点：**
- 電話・メールでの確認に時間がかかる
- 本部・貴社間の二重確認が発生
- 在庫情報の鮮度が不明確
- 納品書作成が手作業
- 関係が薄い店舗は諦めてしまう

### システム導入後の改善

| 課題 | 解決方法 |
|------|---------|
| 7ステップの伝言ゲーム | **4ステップに短縮**：店舗→本部→貴社→加工会社 |
| 本部・貴社間の二重確認 | システムで**最新在庫を一元管理**、確認不要に |
| 在庫確認に時間がかかる | 店舗がLINEで**3秒以内に確認可能** |
| 納品書作成が手作業 | **PDF自動生成**、クラウド保存 |
| 関係が薄い店舗は諦める | **誰でもLINEで気軽に確認・発注**できる |

## 🎯 主要機能（3つの柱）

### 1️⃣ 在庫照会（店舗向け）

**店舗マネージャーがLINEで即座に在庫確認できる**

- 商品を選択して「○○ケースありますか？」と質問
- システムが自動で在庫数を回答（3秒以内）
- 24時間いつでも確認可能

**例：**
> 店舗：「生チョコレート 10ケースありますか？」
> システム：「在庫10ケースあります（※2時間前の情報）」

---

### 2️⃣ 追加発注・承認（店舗・本部）

**店舗が発注 → 本部が承認 → 自動で在庫引当**

- 店舗がLINEで商品と数量を選んで発注依頼
- 本部バイヤーがスマホやPCで承認・差戻し
- **一部承認も可能**：在庫が足りない場合、一部だけ承認できる

**例：一部承認機能**
> 店舗が10ケース依頼、在庫7ケースの場合
> → 本部が7ケース承認
> → 残り3ケースは「在庫待ち」として自動管理

---

### 3️⃣ 最終承認・出荷依頼（貴社 → 加工会社）

**本部承認 → 貴社が最終承認 → 納品書PDF自動生成 → 加工会社に通知**

- 本部承認後、貴社の管理画面に通知が届く
- 貴社が最終承認すると、納品書が自動で作成される
- 加工会社の管理画面に出荷依頼が届く
- 出荷完了を報告すると、店舗に自動通知

**現状のステップ⑦（貴社が納品書作成・納品一覧更新）が自動化**される
